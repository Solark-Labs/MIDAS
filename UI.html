Anonymous Assignment Protocol — Claim C8

1. CONNECT
Platform sends: agent_token, zone, capacity_kg, ETA. No name. No phone. No real ID.
🔒
2. TOKEN
MIDAS stores AGT_TOKEN_[hash] only. Maps to platform for settlement. Never crosses platforms.
⚡
3. SIGNAL
MIDAS sends: {agent_token, order_tokens[]}. Platform dispatches own rider. MIDAS never dispatches.
🤝
4. HANDOVER
SPAN_ID only. Relay rider gets: GPS + masked phone (last 5 digits). Temp session destroyed on delivery.
✅
5. DESTROY
On delivery: session cryptographically destroyed. Secure enclave erased. DPDP 2023 compliant.

THE MINUTES LAYER

MIDAS — Multi-Platform Integrated Delivery Aggregation System — is the intelligence layer that operates between the seconds a delivery order is placed and the minutes it takes to consolidate, route, and dispatch. We make the last mile economically and physically efficient.

The Minutes Layer Framework
The "Minutes Layer" is the temporal window between order placement and dispatch initiation — typically 60–180 seconds depending on city tier. MIDAS occupies this window entirely.
T+0s — ORDER INTAKE
TraceID generated. Weight, zone, tier, platform encoded in 5-field string. O(1) classification in <1ms.
T+2s — PARALLEL MATCH
Simultaneous intra, peer, and cross-tenant matching runs in parallel. Best complement selected via ETA tiebreak.
T+60–180s — DISPATCH
Consolidated batch dispatched. Vehicle utilisation: 85–98%. Up to 38.9% CPO reduction. InvoiceID sealed to TraceID.
Automated Footwear Processing Hardware
MIDAS was originally prototyped on automated conveyor and scanning infrastructure used in footwear manufacturing and dispatch. The patent covers the generalised delivery aggregation layer now deployed across FMCG, QSR, and q-commerce verticals.
QR Scanner Integration
3-check atomic QR handover (Claim C7). Compatible with industrial barcode scanners and mobile QR scanners.
Secure Enclave
Rider device secure enclave for temporary cross-platform session. DPDP 2023 compliant architecture.
GPS Chipset
ETA tiebreak from GPS chipset (Claim C2). Physical hardware input used as deterministic decision variable.
Dispatch Server
City-tier window engine runs on dispatch server. Mode switch causes measurable physical change in network topology.
